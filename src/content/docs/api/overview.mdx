## Base URL

All API requests should be made to:

```
https://api.mygptassistants.com/api/v1
```

## Quick Start <Badge text="Example" variant="success" size="small" />

<Tabs>
  <TabItem label="JavaScript">
    ```javascript
    const response = await fetch('https://api.mygptassistants.com/api/v1/chat/YOUR_BOT_ID/message', {
      method: 'POST',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        message: 'Hello, what can you help me with?'
      }),
    });

    const data = await response.json();
    console.log(data.response);
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    import requests

    response = requests.post(
        'https://api.mygptassistants.com/api/v1/chat/YOUR_BOT_ID/message',
        headers={
            'Authorization': 'Bearer YOUR_API_KEY',
            'Content-Type': 'application/json',
        },
        json={
            'message': 'Hello, what can you help me with?'
        }
    )

    data = response.json()
    print(data['response'])
    ```
  </TabItem>
  <TabItem label="cURL">
    ```bash
    curl -X POST 'https://api.mygptassistants.com/api/v1/chat/YOUR_BOT_ID/message' \
      -H 'Authorization: Bearer YOUR_API_KEY' \
      -H 'Content-Type: application/json' \
      -d '{"message": "Hello, what can you help me with?"}'
    ```
  </TabItem>
</Tabs>

## Rate Limits

<Tabs>
  <TabItem label="Standard Limits">
    | Endpoint Type | Limit |
    |--------------|-------|
    | **Chat endpoints** | 100 requests/minute |
    | **Data endpoints** | 1000 requests/hour |
    | **Webhook endpoints** | 500 requests/minute |
  </TabItem>
  <TabItem label="Rate Limit Headers">
    | Header | Description |
    |--------|-------------|
    | `X-RateLimit-Limit` | Maximum requests allowed |
    | `X-RateLimit-Remaining` | Requests remaining |
    | `X-RateLimit-Reset` | Unix timestamp when limit resets |
  </TabItem>
</Tabs>

<Aside type="note" title="Need Higher Limits?">
Contact support if you need higher rate limits for your use case. Enterprise plans include custom rate limits.
</Aside>

## Error Codes

| Code | Description | Solution |
|------|-------------|----------|
| `RATE_LIMIT_EXCEEDED` | Too many requests | Implement backoff, wait and retry |
| `INVALID_API_KEY` | API key invalid | Check key in dashboard |
| `BOT_NOT_FOUND` | Bot ID doesn't exist | Verify bot ID |
| `INVALID_REQUEST` | Malformed request | Check request body |
| `INTERNAL_ERROR` | Server error | Contact support if persists |

<Aside type="tip" title="Best Practices">
- **Always check the `success` field** before processing data
- **Implement exponential backoff** for rate limit errors
- **Cache responses** when appropriate to reduce API calls
- **Use webhooks** instead of polling for real-time updates
</Aside>
